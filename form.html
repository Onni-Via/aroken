Формы

Тег <form> добавляет на страницу форму, которую пользователь может заполнить. Например, ввести своё имя, фамилию или почту. Данные формы отправляются на сервер. Пример:

<form action="" method="get"> <p> <label for="name">Введите имя:</label> <input type="text" name="name" id="name" required> </p> <p> <label for="email">Введите email:</label> <input type="email" name="email" id="email" required> </p> <button type="submit">Отправить</button> </form>

Атрибуты форм

Базовые атрибуты (стартовый набор)

<action> — здесь указывается ссылка на скрипт, который обработает форму (полный или относительный путь). Если не указать атрибут <action>, то страница будет просто обновляться каждый раз, когда отправляется форма.

<method> — может определять, каким способом будут отправлены на сервер данные, которые ввёл пользователь.

Значения атрибута:

  • get
  • post
  • dialog — значение используется для создания модальных (всплывающих) окон. Но форма должна быть обернута в HTML тег <dialog>.

<name> — уникальное имя формы. Пользователь его не увидит, зато скрипты смогут найти нужную форму.


Атрибут autocomplete

<autocomplete> — атрибут, при помощи которого форма заполняется в один клик — браузер подставит значения в соответствующие поля.

Значения атрибута:

  • on — значение по умолчанию. Включает автозаполнение для этой формы.
  • off — выключает автозаполнение.
  • autocomplete помимо on и off имеет и более узконаправленные значения. Например, autocomplete="tel" и другие категории:
    • Авторизация
    • Формы имени
    • Данные банковской карты и деньги
    • Личные данные
    • Адрес
    • Телефон
    • Ссылки
Форма autocomplete





Кроме элемента <form>, атрибут autocomplete можно указать для любого поля, которое требует ввода данных — <input>, принимающий на вход текст или число, <select> или <textarea>.


Атрибут novalidate

Отключает валидацию формы. Зачем нам валидация формы браузером, когда у нас есть JavaScript?

Атрибут novalidate говорит браузеру не проверять поля и не препятствовать отправке формы, так как иногда подобное поведение бывает нежелательным. К примеру, форма проверяется при помощи JavaScript, и нужно избежать конфликтов с браузерной валидацией. В том числе, и убрать всплывающие подсказки, чтобы показать вместо них свои, кастомные.

input

Тег <input> позволяет создавать интерактивные элементы на сайте — поле для ввода текста, кнопка, ползунок, переключатель и другие.

<input> — это контейнер для интерактивных элементов, с помощью которых пользователь может ввести данные, что-то выбрать, поставить галочку или нажать кнопку.

Если хочется, чтобы введённые или выбранные в <input> данные отправились на сервер, нужно поместить этот тег в <form> либо связать этот элемент с формой через атрибут id.

Разметка input


Если input заключить внутрь label, то отработка будет такая же, как с for и id. Кроме доступности скринридеров (которые читают текст, не проверено, но так сказал ИИ и блогер в ютубе) и могут быть проблемы с динамической генерацией формы (так сказал ИИ).

select и option

Элемент <select> используется, когда нужно показать выпадающий список.

Тег <option>используется в интерактивных элементах управления для вёрстки одиночного пункта списка.

Задать каждому тегу <option> атрибут value. Если этот атрибут не задан, то его значение будет равно текстовому содержимому тега <option>.

selected атрибут указывает на опцию по умолчанию

Внутри тега <select> могут использоваться только теги <option>, <optgroup> и <hr>.









optgroup

Тег <optgroup> позволяет группировать опции внутри элемента <select>. Внутри тега <optgroup> стандартно используется один или несколько тегов <option>.

range

Ползунок (range) - это элемент управления вводом данных, который позволяет пользователю выбирать числовое значение из заданного диапазона. Он представляет собой горизонтальную линию со слайдером, который можно перемещать для выбора значения.

Основные характеристики:

  • Позволяет выбрать только одно значение из указанного диапазона
  • Имеет минимальное и максимальное значение
  • Можно установить шаг изменения значений
  • Часто используется для настройки параметров, где точное значение менее важно, чем его приблизительная величина

Применяется в различных сценариях: регулировка громкости, настройка яркости, фильтры цен, выбор рейтинга и других случаях, когда нужно быстро задать приблизительное значение в определенных пределах.


За отображение числового значения под ползунком в демке отвечает элемент <output>. Но значения из <datalist> сами по себе не отображаются визуально на странице. Они служат только как "подсказки" для элемента <input type="range">, для отображения нужно использовать CSS.


textarea

Тег <textarea> используется для создания многострочного поля ввода с простым текстом. Панели стилизации текста, как в Ворде, являются дополнительными надстройками с использованием JavaScript.

cols - Задаёт ширину поля ввода в символах. сли не задан, то по умолчанию берётся как 20.

rows - Задаёт высоту поля ввода в строках. Если не задан, то по умолчанию высота задаётся равной двум строкам.



<button> VS <input с типом кнопок>


1. <button type="button"> vs <input type="button">

<button type="button">Отправить</button>:

  • Эта кнопка не будет отправлять форму, если не назначить ей обработчик события JavaScript.
  • Может содержать текст, изображения или другие HTML-элементы внутри.
Код: <button type="button">Отправить</button>

<input type="button">:

  • Эта кнопка не будет отправлять форму, если не назначить ей обработчик события JavaScript.
  • Не может содержать вложенные элементы, только текст через атрибут value.
Код: <input type="button" value="Отправить">

2. <button type="submit"> vs <input type="submit">

<button type="submit">Отправить</button>:

  • Это кнопка, которая отправляет форму по умолчанию.
  • Может содержать текст, изображения или другие HTML-элементы внутри.
Код: <button type="submit">Отправить</button>
Отработка кода:

<input type="submit">:

  • Это кнопка, которая также отправляет форму по умолчанию.
  • Не может содержать вложенные элементы, только текст через атрибут value.
Код: <input type="submit" value="Отправить">

3. <button type="reset"> vs <input type="button">

<button type="reset">:

  • Это кнопка, которая сбрасывает значения всех полей формы к их начальным значениям.
  • Может содержать текст, изображения или другие HTML-элементы внутри.
Код:

</button>

  • Не может содержать вложенные элементы, только текст через атрибут value.
